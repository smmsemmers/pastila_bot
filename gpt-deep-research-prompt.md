# Промт для GPT: аудит репозитория + витрина ботов Pastila OS

> **Как использовать:** подключи этот репозиторий к сессии GPT (Deep Research / с доступом к коду),
> вставь весь этот файл как задание. GPT сам прочитает код, подтвердит функции и вернёт
> **готовый красивый промт**, который мы загрузим в Claude Code (бридж) — и он всё реализует.
> Разделы 1–2 — фактура из кода (ground truth для сверки). Раздел 3 — само задание. Раздел 4 — формат вывода.

## РАЗДЕЛ 1. Что за система (ground truth)

Проект **Pastila OS / SEMERS** — бизнес по белёвской пастиле (Лена @elenaisanewleet + Глеб @foxruso).
В связке работают **ТРИ бота** — не путать:

| Бот | Роль | Модель / мозг | Файл в репо | Онлайн |
|---|---|---|---|---|
| 🤖 **@PastilaTaskBot** | Операционный: задачи, напоминания, разбор файлов/фото/голоса, база знаний, Notion | Роутер OpenRouter (`llm_router.py`) — модель под задачу | `bot.py` | Всегда (Render) |
| 💬 **@pastila_gPT_remote_bot** | GPT-чат, поиск, OCR, Codex | Фикс `gpt-5.5` (env `OPENAI_MODEL`) | `~/pastila-gpt-remote/index.mjs` (Node.js) | По требованию |
| 💻 **@pastila_code_remote_bot (БРИДЖ)** | Claude Code: код, деплой, файлы с диска, экспорты, тяжёлое, действия на Маке | Claude (сам Claude Code, не через роутер) | `CLAUDE.md` + скрипты | По требованию |

**Новые решения команды (обязательно отразить в выводе):**
- 🔑 **Бридж @pastila_code_remote_bot = ОСНОВНОЙ бот общения.** Он отвечает первым на всё
  (вопросы, файлы, анализ), таск-бот — только свои слэш-команды и расписания (`BRIDGE_PRIMARY=true`).
- 🔑 **Модель бриджа (Claude): дефолт = Sonnet**, с возможностью быстро переключиться
  на **Fable 5** и **Opus 4.8** (в 1 тап / короткой командой). Это надо описать как механику.
- Модели остальных ботов **оставляем как есть** (уже проанализированы) — задача не менять их,
  а красиво и понятно описать + добавить недостающие «человеческие» команды.

## РАЗДЕЛ 2. Что уже есть в коде (сверь и уточни по факту)

- **Фон (без команды):** голос (Whisper→роутер), файлы (разбор + 10 кнопок), детектор инсайтов, NL-понимание, импорт истории (`result.json`), расписания дайджест/алерты.

### @pastila_gPT_remote_bot (`index.mjs`) — команды
`/gpt [текст]` · `/ocr` (vision-OCR) · `/codex [задача]` (если `ENABLE_CODEX`) · `/status` · `/start` `/help` `/i`
⚠️ В справке таск-бота этот бот описан устаревше как «/research, /agent, OCR» — фактические команды выше. Сверь по коду.

### Бридж @pastila_code_remote_bot (`CLAUDE.md`) — не слэши, а естественный язык
Ключевые слова над файлом: `разбери`/`analyze`, `саммари`, `план`, `стратегия`, `swot`, `приоритеты`, `задачи`, `риски`, `цифры`, `советы`, список сессий. Кнопки через `send-buttons.sh`. Коннекторы: Notion, Google Drive, Gmail, Google Calendar, Asana.

### Модели по задачам (текущие, НЕ менять — только описать)
Общение — Opus 4.8 · voice_route/analyze — GPT-4o mini · plan — Sonnet 4.6 · разбор файлов/strategy/deep — Opus 4.8 · vision — Haiku 4.5 · squeeze — Sonnet 4.6 · речь — Whisper.

## РАЗДЕЛ 3. ЗАДАНИЕ ДЛЯ GPT

Ты — эксперт по продуктовому UX Telegram-ботов и по тому, как объяснять сложные ИИ-инструменты
простым людям. У тебя **подключён репозиторий** — используй его как источник истины.

Сделай следующее:

**1. Проверь и подтверди функции по коду.** Пройди по `bot.py`, `llm_router.py`, `index.mjs`,
`CLAUDE.md`, `README.md`. Для КАЖДОГО из трёх ботов выведи **выверенный** список: что реально
умеет (команды + фоновые функции), какая модель за это отвечает. Если в коде есть функции,
которых нет в README/справке — отметь их (нашёл лишнее / устаревшее описание).

**2. Опиши каждый бот красиво и по-человечески.** Для каждого бота — короткий «портрет»:
- в одну фразу: кто он и зачем;
- 3–5 буллетов «что крутого даёт» на языке пользы (не «есть команда /new», а «за 30 сек заводишь
  задачу, и она сама попадает и в чат, и в таблицу, и напомнит о дедлайне»);
- почему это **ноу-хау** — чем связка трёх ботов сильнее обычного чат-бота.

**3. Придумай для КАЖДОГО бота две новые команды** (тексты — готовые, чтобы вставить в код):
- **`/welcome`** — тёплое приветствие: кто этот бот, с чего начать, 3 примера первых действий.
- **команда-витрина** (предложи имя: напр. `/power`, `/wow` или `/about`) — понятно объясняет:
  *что с помощью этого бота можно улучшить*, *что крутого получить*, *почему это ноу-хау*.
  Тон — вдохновляющий, но конкретный, с эмодзи и примерами. Разный текст под каждого бота.

**4. Опиши механику моделей бриджа (Claude).** Дефолт = **Sonnet**, быстрый свитч на **Fable 5**
и **Opus 4.8**. Предложи: как это переключать (команда/кнопки в 1 тап), как показывать текущую
модель, когда какую советовать (Sonnet — рабочая лошадка, Opus 4.8 — сложное/точность,
Fable 5 — быстро/живо). Учти, что бридж — теперь ОСНОВНОЙ бот общения.

**5. Модели остальных — не трогай**, но сведи в одну наглядную табличку «функция → модель → зачем»,
чтобы человек понимал, кто на чём работает.

## РАЗДЕЛ 4. ФОРМАТ ВЫВОДА (это самое важное)

Верни **ОДИН готовый промт для Claude Code (бриджа)** — красиво оформленный Markdown, который
Лена просто загрузит в Клод, и он реализует. В нём должно быть:

1. **Краткое резюме** (что делаем и зачем) — 3–4 строки.
2. **Выверенные описания трёх ботов** (портреты из п.2).
3. **Готовые тексты новых команд** для каждого бота: `/welcome` и команда-витрина — прямо
   финальным текстом, который пойдёт в код (по-русски, с эмодзи, без плейсхолдеров).
4. **Точечное ТЗ на реализацию для Клода**: в какие файлы что добавить —
   - в `bot.py` — хендлеры `/welcome` и витрины для @PastilaTaskBot;
   - в `index.mjs` — то же для @pastila_gPT_remote_bot;
   - в `CLAUDE.md` — правила `/welcome` и витрины + механику моделей бриджа (Sonnet дефолт, свитч Fable/Opus);
   - в `README.md` — обновить архитектуру (бридж = основной бот общения) и список команд.
5. **Таблица моделей** «функция → модель → зачем» (модели не меняем).
6. **Чек-лист**: что проверить после внедрения.

Требования к стилю: очень чисто, структурно, с таблицами и эмодзи, без воды и без «воды-плейсхолдеров».
Всё по-русски. Каждая новая команда — с готовым текстом, а не «здесь будет приветствие».
