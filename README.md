# 🍬 Pastila OS — Task Bot

Бот для заведения задач через диалог с кнопками. Постит готовый таск в нужный топик группы и пишет строку в Google-таблицу.

## 🤖 Три бота проекта (архитектура)

В связке работают ТРИ бота — не путать:

| Бот | Роль | Мозг / модель | Когда онлайн |
|---|---|---|---|
| 🤖 **@PastilaTaskBot** | Операционный: задачи, напоминания, разбор присланных файлов/фото/голоса кнопками, база знаний, Notion | Модели через **OpenRouter-роутер** (`llm_router.py`); дефолт общения — Opus 4.8, разбор файлов/стратегия — Opus 4.8, план — Sonnet 4.6, голос/analyze — GPT-4o mini | Всегда онлайн (Render) |
| 💬 **@pastila_gPT_remote_bot** | Чат и поиск: `/research`, `/agent` (глубокое исследование в интернете), OCR | GPT | По требованию |
| 💻 **@pastila_code_remote_bot** | Claude Code: код, деплой, файлы с диска, экспорты, тяжёлое, действия на Маке | **Claude Opus 4.8 (1M)** — сам Claude Code, НЕ через роутер | По требованию (запуск `start-code-bridge.command`) |

**BRIDGE_PRIMARY** (env, по умолчанию `true`): когда бридж @pastila_code_remote_bot главный — task-бот НЕ лезет сам в голос / свободный текст / файлы / инсайты (чтобы не перебивать бридж), но остаётся на свои команды и расписания. Функции ниже, помеченные 🔇, в этом режиме выключены у task-бота.

## 📋 Все функции @PastilaTaskBot (полный список из кода)

### Задачи
- `/new` — завести задачу диалогом с кнопками (название → DoD → кто → дедлайн → шаги → материалы → теги → статус). Постит в топик + пишет строку в Google Sheets.
- `/menu` — панель быстрых действий (приоритет, план, статусы) с выбором человека.
- `/list` — открытые задачи (статус ≠ DONE/CANCELLED), сгруппированы по исполнителю.
- `/status` — реплаем на задачу: сменить статус (в таблице и в сообщении). Плюс кнопки под задачей: 🔵 В работу / 🟣 Ревью / ✅ Done.
- `/edit` — реплаем на карточку задачи: изменить любое поле.
- `/digest` — дайджест дедлайнов на сегодня (ручной запуск; авто — по расписанию).
- `/alerts` — алерты по дедлайнам на завтра (ручной запуск).
- `/recurring` — повторяющиеся задачи: `/recurring [задача] — [кто] — каждые [N] дней` | `list` | `del [N]`.
- `/remind_when` — триггерное напоминание: `/remind_when [ключевое слово] — [что напомнить]`.
- `/purge` — удалить ВСЕ задачи из таблицы (с подтверждением).

### LLM-аналитика (нужен OPENROUTER_API_KEY)
- `/analyze` — 🔇 разобрать накопленную переписку → предложить задачи-черновики. Модель: `analyze` (GPT-4o mini).
- `/plan` — план работы отдельно для Лены и Глеба по переписке (+ контекст после команды). Модель: `plan` (Sonnet 4.6).
- `/strategy` — стратегический анализ «что делать дальше» на Opus 4.8 + extended thinking.
- `/deep` — двухэтапно: Perplexity ищет в интернете → Opus синтезирует с внутренними данными.
- `/post` — черновик поста для соцсетей из идеи.
- `/q` (`/quick_idea`) — мгновенно сохранить идею в базу знаний без диалога.
- `/dash` — сводка: задачи по людям, просрочены, KB, последняя активность.
- `/pin` — интерактивное описание группы с навигацией по разделам.

### База знаний и интеграции
- `/session [название]` — начать запись сессии/встречи/лога (текст/файл/голос) в базу знаний.
- `/kb` — показать, что сейчас в базе знаний.
- `/find [запрос/#тег]` — поиск по базе знаний.
- `/notion sync` — синхронизировать страницы из Notion в базу знаний.

### Модели и сервис
- `/model` — модель для общения (сводка активных моделей по задачам); `/model all` — полная таблица всех моделей; `/model [задача]` — выбор кнопками (общение/план/разбор/голос).
- `/fable` — быстрый тумблер общения: Claude Fable 5 ↔ Opus 4.8.
- `/trim` — уровень обрезки контекста 0–10 / режим `truncate`|`squeeze` (умное сжатие).
- `/tokens` — статистика расхода токенов и стоимость с запуска.
- `/ai` — проверка ключей: OpenAI (Whisper) и OpenRouter (LLM).
- `/id` — chat_id и id топика (для настройки).
- `/start`, `/help`, `/welcome` — справка/приветствие.
- `/cancel` — отменить создание задачи.

### Автоматические (фоновые) функции — без явной команды
- 🔇 **Голос** — Whisper (env `OPENAI_MODEL`) расшифровывает → роутер (`voice_route`, GPT-4o mini) определяет тип → задача/список/план/analyze/ответ. Длинное голосовое (>50 слов) → структурированный разбор.
- 🔇 **Файлы** — PDF/DOCX/таблица/текст/изображение/аудио: разбор на Opus 4.8 (`_ingest_analyze`) + 10 кнопок анализа под файлом (Разбор/Саммари/План/Стратегия/SWOT/Приоритеты/Задачи/Риски/Цифры/Советы). Vision-описание картинок — `VISION_MODEL` (Haiku 4.5).
- 🔇 **Детектор инсайтов** — авто-реакция ⭐ + теги + сохранение ценных мыслей в KB.
- 🔇 **NL-понимание** — понимает просьбы словами («поставь встречу», «покажи задачи», «напомни») → действие.
- **Импорт истории** — пришли боту `result.json` (Telegram Desktop → Export chat history → Machine-readable JSON) → загрузит прошлую переписку в память (для /plan, /analyze). Экспорты Claude (`data-*.zip`, `conversations.json`) маршрутизируются на бридж.
- **Расписания** — авто-дайджест дедлайнов и алерты по времени.
- **Приветствие** при добавлении в группу.

> 🔇 = выключено у task-бота, когда `BRIDGE_PRIMARY=true` (эти функции ведёт бридж @pastila_code_remote_bot).

## 💬 Функции @pastila_gPT_remote_bot (код: `~/pastila-gpt-remote/index.mjs`, Node.js)

Лёгкий GPT-мост. Модель: **gpt-5.5** (env `OPENAI_MODEL`). Команды из кода:
- `/gpt [текст]` — спросить GPT. Также отвечает на упоминание, реплай боту, в личке и на присланные фото/документы.
- `/ocr + картинка` — извлечь текст с картинки (OCR через vision). Можно подписью к фото или реплаем на картинку.
- `/codex [задача]` — запустить Codex CLI, если включён (`ENABLE_CODEX`); работает в `CODEX_WORKDIR` с песочницей `CODEX_SANDBOX`.
- `/status` — user_id, chat_id, тип чата, текущая модель, статус Codex.
- `/start` `/help` `/i` — справка.

> Примечание: в справке task-бота этот бот упомянут как «/research, /agent, OCR» — это устаревшее описание; фактические команды в коде — `/gpt`, `/ocr`, `/codex`, `/status`.

## 💻 Функции @pastila_code_remote_bot — бридж (Claude Code, Opus 4.8 1M)

Не слэш-команды, а естественный язык + правила из `CLAUDE.md`. Запуск: `start-code-bridge.command`. Умеет:
- **Тяжёлое:** экспорты Claude (`conversations.json`, `data-*.zip`) по сессиям, файлы с диска, ZIP, код, деплой на Render, действия на Маке.
- **Ключевые слова над файлом/материалом:** `разбери`/`/analyze`, `саммари`, `план`, `стратегия`, `сильные-слабые`(swot), `приоритеты`, `задачи`, `риски`, `цифры`, `советы`, список сессий.
- **Кнопки** при получении файла (кликабельные, через `send-buttons.sh`).
- **Коннекторы (через claude.ai):** Notion, Google Drive, Gmail, Google Calendar, Asana — читать/доставать документы, письма, страницы.
- **Прогресс** длинных задач через `reply` + `edit_message`.
- Заведение задач в общую Google-таблицу команды (подхватывает task-бот).
- Отвечает ТОЛЬКО через инструмент `reply` (терминал в чат не попадает).

## Что делает

1. Команда `/new` запускает диалог
2. Бот по шагам спрашивает: название → DoD → кто → дедлайн → шаги → материалы → теги → статус
3. Собирает красивый таск по шаблону Pastila OS
4. Постит в топик: Лена → Tasks—Лена, Глеб → Tasks—Глеб, Лена+Глеб → оба
5. Тегает исполнителя (@elenaisanewleet / @foxruso)
6. Пишет строку в Google Sheets (в колонку «Ссылка» кладёт deep-link на опубликованную задачу)

## Команды

- `/new` — завести новую задачу (диалог с кнопками)
- `/list` — показать открытые задачи (статус ≠ DONE/CANCELLED), сгруппированные по исполнителю
- `/status` — сменить статус задачи: ответь этой командой на сообщение с задачей и выбери новый статус. Бот обновит строку в таблице (по названию задачи) и само сообщение
- `/digest` — вручную прислать дайджест дедлайнов на сегодня (для проверки)
- `/alerts` — вручную разослать алерты по дедлайнам на завтра (для проверки)
- `/analyze` — разобрать накопленную переписку (с момента запуска бота) и предложить задачи-черновики. Нужен `OPENAI_API_KEY`. Бот видит сообщения только начиная с момента, как его добавили (ограничение Telegram), и держит их в памяти до перезапуска
- `/plan` — составить конкретный план работы отдельно для Лены и для Глеба по переписке. Можно добавить контекст после команды: `/plan что успеть к запуску`. Нужен `OPENAI_API_KEY`

**Импорт истории.** Бот видит сообщения только с момента запуска. Чтобы он учитывал и **прошлую** переписку — экспортируй чат в Telegram Desktop (⋮ → Export chat history → формат **Machine-readable JSON**) и **пришли боту файл `result.json`**. Бот загрузит историю в память, и `/plan`, `/analyze` и голосовой поиск начнут её учитывать (хранится до перезапуска бота).
- `/id` — показать `chat_id` и id топика прямо в чате (для сбора переменных при настройке)
- `/cancel` — отменить создание задачи
- `/start` — короткая справка

Под каждой опубликованной задачей есть кнопки быстрой смены статуса — `🔵 В работу`, `🟣 Ревью`, `✅ Done` — меняют статус в один тап (и в таблице, и в самом сообщении). Для остальных статусов используй `/status` реплаем.

### Голосовой ввод

Пришли боту **голосовое** — бот распознаёт речь (OpenAI Whisper) и сам понимает намерение:
- **задача** («нужно купить коробки до пятницы») → разбирает на поля и показывает черновик с выбором статуса;
- **поиск по переписке** («найди, где Глеб говорил про сроки», «что решили по пастиле») → ищет ответ в накопленной переписке и отвечает цитатами.

Поиск работает по сообщениям, которые бот видел с момента запуска (как `/analyze`).

Нужен ключ OpenAI в переменной `OPENAI_API_KEY` (без него фича просто выключена). Необязательные: `OPENAI_MODEL` (по умолчанию `gpt-4o-mini`), `WHISPER_MODEL` (по умолчанию `whisper-1`). Стоит копейки за сообщение.

### Утренний дайджест

Каждый день в **12:00** бот шлёт в General список задач с дедлайном = сегодня и тегает Лену и Глеба. Если на сегодня дедлайнов нет — молчит. Проверить вручную в любой момент — командой `/digest`.

Настраивается переменными окружения (необязательные):
- `DIGEST_HOUR` — час отправки (по умолчанию `12`)
- `DIGEST_MINUTE` — минута (по умолчанию `0`)
- `TZ` — таймзона (по умолчанию `Europe/Moscow`)

⚠️ **Важно:** дайджест шлётся, только пока воркер не «спит». На бесплатном плане Render воркер засыпает при простое — тогда дайджест может не прийти вовремя. Для гарантии нужен платный план или keep-alive.

### Дедлайн-алерты

Раз в день бот находит задачи с дедлайном = **завтра** и тегает исполнителя **в его топике** (Лена → топик Лены, Глеб → топик Глеба, «Лена + Глеб» → в оба). Проверить вручную — `/alerts`.

Настройка (необязательные переменные): `ALERT_HOUR` (по умолчанию `12`), `ALERT_MINUTE` (`0`); таймзона общая с дайджестом — `TZ`. Тот же нюанс про засыпающий воркер.

## Перед деплоем нужно собрать 8 переменных

### 1. BOT_TOKEN
Токен твоего бота от @BotFather — это **PastilaTaskBot** (`@createtaskpastila_bot`).
В @BotFather: выбери бота → API Token → Copy. Этот токен идёт только в переменную
`BOT_TOKEN` на Render, в код и в гит его не кладём.

### 2-4. GROUP_CHAT_ID, THREAD_LENA, THREAD_GLEB

Это id группы и id топиков.

**Самый простой способ — через сам бот (команда `/id`):**
- задеплой бота хотя бы с одним `BOT_TOKEN` (остальное можно добавить позже — бот не упадёт),
- добавь его в группу, сделай админом,
- зайди в топик Лены, напиши `/id` → бот пришлёт `GROUP_CHAT_ID` и id этого топика (это `THREAD_LENA`),
- повтори в топике Глеба → получишь `THREAD_GLEB`,
- впиши значения в Render и передеплой.

**Либо через стороннего бота:**

**Способ через @getidsbot или @RawDataBot:**
- Добавь @RawDataBot в группу на минуту
- Напиши любое сообщение в топик Tasks — Лена
- Бот покажет JSON. Найди в нём:
  - `chat.id` → это GROUP_CHAT_ID (отрицательное, типа `-1001234567890`)
  - `message_thread_id` → это THREAD_LENA для этого топика
- Повтори в топике Tasks — Глеб → получишь THREAD_GLEB
- Удали @RawDataBot

**Альтернатива — из ссылки на топик:**
- Открой топик в веб-версии Telegram, скопируй ссылку вида
  `https://t.me/c/2493586/12` — последнее число `12` это thread_id,
  а `2493586` → GROUP_CHAT_ID будет `-100` + это число = `-1002493586`

**Режим одной группы (проще).** Если не хочешь возиться с топиками — **не задавай** `THREAD_LENA` и `THREAD_GLEB` (оставь пустыми). Тогда бот шлёт все задачи в саму группу `GROUP_CHAT_ID`, а на исполнителя указывает тегом (@elenaisanewleet / @foxruso) в тексте. Файлы, приложенные на шаге «Материалы», бот кладёт сообщением **прямо под задачу** в той же группе.

### 5. SHEET_ID
Из адреса таблицы между `/d/` и `/edit`:
`docs.google.com/spreadsheets/d/`**`ВОТ_ЭТОТ_КУСОК`**`/edit`
У тебя: `1Mlwvuw4bc7ove-2PS20fYybgWlN0rXQnkrfSR7lkVRM`

### 6. SHEET_NAME
Имя листа. У тебя `Sheet1`.

### 7. GOOGLE_CREDENTIALS — сервисный ключ
Это JSON-ключ сервисного аккаунта Google. Получить:
1. console.cloud.google.com → создай проект (или возьми существующий)
2. APIs & Services → Enable APIs → включи **Google Sheets API** и **Google Drive API**
3. Credentials → Create Credentials → **Service Account**
4. Создай аккаунт, зайди в него → вкладка Keys → Add Key → JSON → скачается файл
5. Открой этот JSON, скопируй **всё содержимое одной строкой** → это значение GOOGLE_CREDENTIALS
6. **ВАЖНО:** в JSON есть поле `client_email` (типа `xxx@xxx.iam.gserviceaccount.com`).
   Открой свою Google-таблицу → Настройки доступа → добавь этот email как **Редактор**.
   Иначе бот не сможет писать в таблицу.

### 8. TAG_LENA / TAG_GLEB
Уже прописаны по умолчанию: `@elenaisanewleet` и `@foxruso`.

### 9. OPENROUTER_API_KEY (для /analyze, /plan, /model и разбора голоса)
Ключ с [openrouter.ai](https://openrouter.ai): зарегистрируйся → Keys → Create Key → пополни баланс.
Один ключ открывает 30+ моделей. Без него текстовые LLM-функции просто выключены (бот работает).
Необязательные при этом: `TRIM_LEVEL` (`4`), `CONTEXT_MODE` (`truncate`), `SQUEEZE_MODEL` (`sonnet46`).

`OPENAI_API_KEY` остаётся нужен **только** для распознавания речи (Whisper) — это отдельная фича.

## Деплой на Render

1. Залей этот проект на GitHub (новый репозиторий)
2. Render → New → **Background Worker** (НЕ Web Service! у бота нет порта)
3. Подключи репозиторий
4. Build Command: `pip install -r requirements.txt`
5. Start Command: `python bot.py`
6. В разделе **Environment** добавь переменные выше
   (GOOGLE_CREDENTIALS — весь JSON одной строкой)
7. Deploy

Через минуту бот оживёт. Напиши ему `/new` в личку или в группе.

## Важные нюансы

- **Версия Python.** `python-telegram-bot 21.6` несовместим с Python 3.14 (там `run_polling()` падает с `RuntimeError: There is no current event loop`). Версия закреплена файлом `.python-version` (3.12) и переменной `PYTHON_VERSION`. Если бот в логах Render крутит этот рестарт-цикл — проверь, что используется Python 3.12/3.13, а не 3.14.
- **Privacy mode** бота должен быть выключен (мы это уже сделали в @BotFather).
- Бот должен быть **админом** группы с правом отправки сообщений.
- На бесплатном плане Render воркер может «засыпать» при простое — для постоянной работы держи активность или возьми платный план ($7/мес). Но для редких задач бесплатного хватает.
- Если бот пишет таск, но не пишет в таблицу — почти всегда не выдан доступ сервисному email к таблице (пункт 7, шаг 6).
