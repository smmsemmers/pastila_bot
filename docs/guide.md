# 🤖 Pastila OS — бот для задач. Как работает (с примерами)

Телеграм-бот для задач команды (**Лена** и **Глеб**). Превращает поручения в аккуратные карточки, постит в нужный топик группы, пишет всё в Google-таблицу и напоминает о дедлайнах. Часть фич — на голосе и GPT.

---

## ⚡ Коротко

- Завести задачу: **`/new`** (по шагам) или просто **пришли голосовое**.
- Посмотреть, что в работе: **`/list`**.
- Сменить статус: кнопки под задачей или **`/status`** ответом на неё.
- Бот сам напоминает о дедлайнах — утром и за день до срока.
- Разбирает переписку: **`/analyze`** (задачи) и **`/plan`** (план работы).

---

## 1. Создать задачу — `/new`

Бот ведёт по шагам, на каждом можно нажать **⏭ Пропустить**:

1. **Название** — текстом.
2. **🏁 DoD** — когда задачу считаем сделанной (текст / пропустить).
3. **👤 Кто** — кнопки: `Лена` · `Глеб` · `Лена + Глеб`.
4. **🗓️ Дедлайн** — формат `ДД.ММ` (напр. `25.07`); пропустить → **Backlog**.
5. **📋 Шаги** — каждый с новой строки (текст / пропустить).
6. **📎 Материалы** — текст/ссылка **или пришли файл/фото** (приложу под задачу).
7. **🏷️ Теги** — напр. `#tilda #лендинг` (текст / пропустить).
8. **🚦 Статус** — кнопка → **задача публикуется**.

В конце бот отвечает:

> ✅ Задача создана и отправлена в топик.
> 📊 Записана в таблицу.

## 2. Карточка задачи — пример

Так выглядит опубликованная задача (тег исполнителя подставляется сверху сам):

```
@elenaisanewleet
📌 ЗАДАЧА: Сверстать лендинг акции
🏁 DoD: Открывается на мобиле, кнопка ведёт на оплату
———————————
👤 КТО: Лена
🗓️ ДЕДЛАЙН: 25.07
———————————
📋 ЧТО СДЕЛАТЬ
   ✦ Собрать блоки в Tilda
   ✦ Подключить оплату
   ✦ Проверить на телефоне
———————————
📎 МАТЕРИАЛЫ: 📎 макет.fig (приложен ниже)
🏷️ ТЕГИ: #tilda #лендинг
———————————
🟡 TODO
```

---

## 3. Кнопки под задачей (быстрый статус)

Под каждой карточкой — кнопки в один тап (3 в ряд):

> `🔵 В работу` · `🟠 Ждём` · `🟣 Ревью`
> `🔴 Блок` · `✅ Done`

Нажатие меняет статус **сразу и в карточке, и в таблице**.

| Кнопка | Статус |
|---|---|
| 🔵 В работу | WIP |
| 🟠 Ждём | WAITING |
| 🟣 Ревью | REVIEW |
| 🔴 Блок | BLOCKED |
| ✅ Done | DONE |

---

## 4. Сменить статус — `/status`

Ответь (reply) командой **`/status`** на сообщение с задачей — бот покажет все 8 статусов кнопками и обновит и текст задачи, и строку в таблице.

---

## 5. Список открытых — `/list`

Показывает всё, кроме `DONE` и `CANCELLED`, сгруппировано по людям:

```
📋 ОТКРЫТЫЕ ЗАДАЧИ

👤 Лена — 2
   🟡 TODO  Сверстать лендинг акции  ·  🗓️ 25.07
   🔵 WIP  Тексты для рассылки  ·  🗓️ Backlog
👤 Глеб — 1
   🟠 WAITING  Договор с подрядчиком  ·  🗓️ 28.07

Итого открытых: 3
```

## 6. 🎙️ Голос — два режима

Просто пришли боту **голосовое**. Он распознаёт речь и сам понимает намерение:

**а) Завести задачу.** «*Лене — свёрстать лендинг акции к 25 июля*»:

```
🎙️ Услышал: «Лене свёрстать лендинг акции к 25 июля»

— Черновик —
@elenaisanewleet
📌 ЗАДАЧА: Сверстать лендинг акции
👤 КТО: Лена
🗓️ ДЕДЛАЙН: 25.07
...
🚦 Выбери статус — и задача опубликуется:
```

Выбираешь статус кнопкой (или ❌ Отмена) — публикуется как обычная задача.

**б) Спросить по переписке.** «*найди, где Глеб говорил про сроки оплаты*» — бот **не заводит задачу**, а ищет ответ в истории чата:

> 🔎 Глеб писал 14.06: «оплату подрядчику нужно закрыть до 28-го»…

---

## 7. 🧠 Задачи из переписки — `/analyze`

Бот тихо копит сообщения чата. По команде `/analyze` отдаёт их GPT и предлагает **до 5 готовых черновиков** задач (особенно то, что просит Глеб). Каждый черновик — с кнопками статуса: жмёшь — заводится.

---

## 8. 🗂 План работы — `/plan`

Составляет конкретный план по переписке, отдельно для каждого:

```
📋 План для Лены:
1. …
2. …

📋 План для Глеба:
1. …
```

Можно добавить контекст прямо в команду: `/plan что нужно успеть к запуску`.

---

## 9. 📚 Импорт истории — файл `result.json`

В Telegram: **Export chat history → формат Machine-readable JSON**. Пришли файл боту — он загрузит прошлую переписку в память, и `/plan`, `/analyze`, голосовой поиск начнут её учитывать.

> 📚 Загрузил историю: 1240 сообщений…

## 10. ⏰ Напоминания о дедлайнах

По расписанию (по умолчанию **12:00 МСК**). Те же команды можно вызвать вручную в любой момент.

**`/digest`** — дедлайны на **сегодня**, в общий чат:

```
☀️ Доброе утро! Дедлайн сегодня — 21.06:

   🟡 TODO  Сверстать лендинг акции — 👤 Лена
   🟠 WAITING  Договор с подрядчиком — 👤 Глеб

@elenaisanewleet @foxruso
```

**`/alerts`** — задачи с дедлайном **завтра**, каждому в его топик с тегом:

```
@elenaisanewleet
⏰ Дедлайн завтра (22.06)!
🟡 TODO  Сверстать лендинг акции
```

## 11. Служебные команды

| Команда | Что делает |
|---|---|
| `/new` | создать задачу по шагам |
| `/list` | открытые задачи |
| `/status` | сменить статус (ответом на задачу) |
| `/digest` | дедлайны на сегодня |
| `/alerts` | дедлайны на завтра |
| `/analyze` | найти задачи в переписке |
| `/plan` | план работы для Лены и Глеба |
| `/id` | chat_id и id топика (для настройки) |
| `/cancel` | отменить создание задачи |
| `/start`, `/help` | что умеет бот |

## 📊 Куда всё попадает

**Google-таблица** — по строке на задачу:

| Колонка | Что |
|---|---|
| Дата | дата создания (ГГГГ-ММ-ДД) |
| Кто | Лена / Глеб / Лена + Глеб |
| Задача | название |
| Дедлайн | `ДД.ММ` или `Backlog` |
| Статус | текущий статус |
| Ссылка | прямой линк на сообщение задачи в группе |

**Маршрутизация постов:**

- Заданы топики (`THREAD_LENA`/`THREAD_GLEB`) → задача летит в топик исполнителя; «Лена + Глеб» → в оба.
- Топиков нет → одна группа, исполнитель виден по тегу в тексте.

## 🚦 Статусы

| | Код | Когда |
|---|---|---|
| ⚪️ | NEW | только заведена, ещё не в работе |
| 🟡 | TODO | взято в план |
| 🔵 | WIP | в работе |
| 🟠 | WAITING | ждём ответа / материалов |
| 🟣 | REVIEW | на проверке |
| 🟢 | DONE | готово · *скрыто из `/list`* |
| 🔴 | BLOCKED | заблокировано |
| ⚫️ | CANCELLED | отменено · *скрыто из `/list`* |

## ⚠️ Что важно помнить

- **Голос, `/analyze`, `/plan` работают только с ключом `OPENAI_API_KEY`.** Без него голосовые молча игнорируются, а `/analyze`/`/plan` отвечают «выключено».
- **Переписка хранится в памяти и обнуляется при перезапуске бота** (держим последние ~2000 сообщений на чат). На бесплатном плане Render воркер «засыпает» в простое → при пробуждении история стирается, а авто-дайджест/алерты по расписанию могут не сработать. Для надёжного 24/7 — план **Starter**. Файл `result.json` — способ быстро вернуть историю.
- **Дедлайн** понимается только как `ДД.ММ` (разделители `.`, `-`, `/`); что не распознал — уходит в **Backlog**.
