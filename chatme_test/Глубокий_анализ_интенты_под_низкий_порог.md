# Глубокий анализ: интенты и обучающие фразы под НИЗКИЙ порог

Цель: сделать так, чтобы можно было опустить CONFIDENCE THRESHOLD (≈40–50) и бот всё равно
работал корректно — не отвечал уверенно не на тот вопрос и не тащил мусор в реальные интенты.

---

## 0. Ключевая идея (почему низкий порог вообще может быть безопасным)

NLU — классификатор закрытого набора: на любой ввод он раскидывает 100% уверенности между твоими
интентами. Порог лишь решает «распознал / Fallback». Значит низкий порог безопасен ТОЛЬКО если:

1. **Реальные интенты хорошо разделены** — у похожих тем разный словарь, чтобы верный интент
   выигрывал с большим отрывом (70–90%), а не «45 против 43».
2. **У мусора есть свой сильный аттрактор** — интент «Не по теме», куда стекается вся масса
   уверенности на гиббериш/офф-топик. Тогда на реальных интентах мусор набирает мало.

Если эти два условия выполнены — порог 40–50 работает: верные вопросы проходят, мусор уходит в
«Не по теме» (вежливый отказ), а Fallback ловит только реально пограничное. **Понижать порог без
разделения интентов — как раз то, что даст «уверенные ошибки». Поэтому сначала разделение, потом
порог.**

## 1. Карта «путающихся» пар и что я развела

| Пара интентов | Из-за чего путались | Что сделала |
|---|---|---|
| **Оформление ↔ Оплата** | фразы «способ оплаты при оформлении» | убрала «оплату» из Оформления; Оформление = только про сбор/создание заказа |
| **Отмена ↔ Возврат** | обе про «деньги/отказ» | Отмена = ДО отправки (отменить/аннулировать); Возврат = ПОСЛЕ получения (спор/дефект/вернуть товар) |
| **Возврат ↔ Получение** | «полученный заказ» | перефразировала в «уже пришедший товар», убрала слово-триггер |
| **Отмена ↔ Оформление** | «оформленный заказ» | → «уже созданный заказ» |
| **Возможности ↔ Поддержка** | общее слово «вопрос» | Поддержка = явно про ЖИВОГО оператора; Возможности = про темы бота |
| **Отслеживание ↔ Получение** | «где заказ» vs «получить» | оставила: разные глаголы (где/отследить vs получить/забрать/код) |
| **Погода ↔ Как дела** | оба на «как…» | разделены по смыслу, ок |

Принцип разделения: **у каждого интента — свой якорный глагол/существительное.**
Оформить · Оплатить · Отследить · Получить/забрать/код · Отменить/аннулировать · Вернуть/спор ·
Оператор/поддержка. Чем меньше пересечений слов — тем выше отрыв верного интента → тем ниже можно
опустить порог.

## 3. Как подобрать порог ЭМПИРИЧЕСКИ (док-метод, не на глаз)

1. Привяжи «Не по теме» подслотом к NLU → **TRAIN**.
2. Прогони два батча и смотри колонку CONFIDENCE в Analytics → SHOW REPORTS:
   - **Батч A (реальные вопросы, по 2–3 на интент)** — запиши минимальную уверенность ВЕРНОГО
     ответа. Обычно 70–90%.
   - **Батч B (новый мусор + офф-топик, которого НЕТ в обучении)** — запиши максимальную
     уверенность, которую мусор набирает НА РЕАЛЬНОМ интенте (не на «Не по теме»).
3. Ставь порог **между** этими числами. Практический ориентир: **45–50**.
   - Ниже минимума верных ответов → не режешь живые вопросы.
   - Выше «протечки» мусора в реальные интенты → мусор туда не проходит.

⚠️ **Не опускай ниже ~40.** На совсем низком пороге похожие интенты (Отмена/Возврат) начнут
отвечать уверенно-неправильно вместо «переформулируйте». 45–50 — золотая середина: и порог низкий,
как ты хотела, и ошибок нет.

## 4. Чек-лист внедрения
1. Замени фразы во всех интентах на списки из раздела 2.
2. Убедись, что «Не по теме» **привязан как подслот** к NLU «Распознавание вопроса».
3. Убери двойное приветствие (интент «Приветствие ещё одно» дублирует welcome от Start).
4. Ответ «Благодарности» смягчи (не «До свидания»).
5. **TRAIN.**
6. Прогони батчи A и B, поставь порог 45–50 по разделу 3.
7. Выключи «Сбрасывать состояние общения» + подними «Таймаут неактивности» (настройки агента).
