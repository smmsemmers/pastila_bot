## 1. Этапы, сроки, бюджет

Цены — **ориентир** (RU-рынок 2026), под «экономно, но разумно». Финал — после подтверждения границ и модели оплаты (фикс за этап).

| Этап | Что делаем | Срок | Ориентир, ₽ |
|------|-----------|------|-------------|
| **0. Kick-off / доступы** | Скоуп, тест-учётки, read-only код, правила. | 1–2 дня | 0 (входит) |
| **1. Внешний периметр + автопроверки** | Скан периметра, конфиги, information exposure; **настройка SAST/DAST/dep-scan в CI** (чтобы дальше ловилось само). | 3–4 дня | 60–90k |
| **2. Веб + API (ядро)** | my.perimetr.one + API по OWASP WSTG/ASVS/API Top 10: authn/authz, **IDOR/BOLA**, инъекции, rate-limit, бизнес-логика; PDF через Puppeteer → **SSRF**. | 6–8 дней | 150–220k |
| **3. Серверная логика + инфра** | NestJS: сессии/токены, RBAC, инъекции (Drizzle/pg), Redis, файловые операции MinIO/S3, конфиг Selectel. | 3–5 дней | 80–130k |
| **4. Мобилки (iOS/Android)** | OWASP MASVS/MASTG: Capacitor/WebView, хранение на устройстве, трафик. | 4–5 дней | 90–140k |
| **5. Отчёт + приоритизация** | Отчёт по CVSS v4.0/CWE, приоритизированный план фиксов, дефенс-брифинг команде. | 2–3 дня | 40–60k |
| **6. Ретест (опц.)** | Проверка закрытых уязвимостей после ваших правок. | 2–3 дня | 50–80k |

**Итого:**
- **Полный объём (этапы 0–5):** ~**420–640k ₽**, ~**4–5 недель**.
- **Ретест (этап 6):** +50–80k, +2–3 дня.

## 2. Экономный сценарий (рекомендую для старта)

Раз главное — **не пустить дыры в прод перед запуском**, а данных/платежей ещё нет, можно **сузить первый заход** до самого критичного и добрать остальное позже:

**Фаза A (must-have до запуска):** Этапы 0 + 1 + 2 + 5
→ веб/API/периметр + автопроверки + отчёт.
→ ~**250–370k ₽**, ~**2–3 недели.**

**Фаза B (после запуска / по мере роста):** Этапы 3 + 4 (+ 6 ретест)
→ инфра, мобилки, ретест — когда появятся клиенты и биллинг.

Так вы закрываете 80% риска за ~половину бюджета и не тормозите запуск.

## 3. Что вы получаете

- Приоритизированный список уязвимостей (CVSS v4.0 + CWE), с шагами воспроизведения и рекомендациями.
- Настроенные автопроверки в CI (чтобы регрессии ловились без нас).
- Финальный отчёт + короткий брифинг для команды разработки.
- (Опц.) Ретест-подтверждение закрытия.
