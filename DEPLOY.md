# 🚀 Деплой ботов на Render

Гайд, как поднять ботов на [Render](https://render.com) и держать их включёнными.
Здесь два бота:

| Бот | Что это | Где код |
|-----|---------|---------|
| **@PastilaTaskBot** (`@createtaskpastila_bot`) | Основной бот: задачи, база знаний, ИИ-анализ | Этот репозиторий (`bot.py`) |
| **@pastila_code_remote_bot** | Claude Code Remote — пишет код и деплоит фичи | Отдельно (см. [ниже](#-второй-бот--pastila_code_remote_bot)) |

---

## ⚠️ Сначала — честно про «чтобы всегда работали» (24/7)

Бот `bot.py` работает через **long-polling** (`run_polling`): он сам ходит в Telegram
за обновлениями, входящего HTTP-порта у него нет. Из этого вытекает главное про план Render:

- **Фоновых воркеров на бесплатном плане Render больше нет.** Worker доступен только на
  платных планах — минимум **Starter (~$7/мес за сервис)**.
- **Бесплатный web-сервис засыпает** через 15 минут простоя (холодный старт 30–60 сек).
  Пока он спит — polling не идёт, и **дайджест/алерты по расписанию не сработают**.
- **Лимит бесплатного плана — 750 часов/мес на весь аккаунт.** Два сервиса, работающие
  круглосуточно, — это ~1440 часов, лимит превышается, и **все** бесплатные сервисы
  блокируются до конца месяца.
- Render **не рекомендует** self-ping (пинговать самого себя, чтобы не заснуть) — за это
  можно словить блокировку.

**Вывод:** для реального 24/7 нужен платный worker на каждого бота (два бота ≈ $14/мес).
Это единственный надёжный способ. Бесплатный вариант ниже — только как временный, с оговорками.

## 🤖 Бот 1: @PastilaTaskBot (этот репозиторий)

### Шаг 1. Код уже на GitHub

Репозиторий: `smmsemmers/pastila_bot`. Ветка деплоя настраивается в Render (обычно `main`).
Пуш в эту ветку → Render передеплоит автоматически.

### Шаг 2. Собери переменные окружения

Минимум для запуска — только `BOT_TOKEN` (бот не упадёт без остальных, но часть функций выключится).
Полный список и как их получить — в [README.md](README.md#перед-деплоем-нужно-собрать-8-переменных).

Обязательные для полноценной работы:

| Переменная | Что это |
|------------|---------|
| `BOT_TOKEN` | токен от @BotFather |
| `GROUP_CHAT_ID` | id группы (можно узнать командой `/id` в чате) |
| `THREAD_LENA`, `THREAD_GLEB` | id топиков (или оставить пустыми — режим одной группы) |
| `SHEET_ID`, `SHEET_NAME` | Google-таблица задач |
| `GOOGLE_CREDENTIALS` | JSON сервисного аккаунта **одной строкой** |

Опциональные (включают ИИ-функции): `OPENAI_API_KEY` (голос/Whisper),
`OPENROUTER_API_KEY` (`/analyze`, `/plan`, `/model`), `NOTION_TOKEN` и др. — см. `render.yaml`.

> 🔒 Токены и `GOOGLE_CREDENTIALS` **никогда не коммить в git** — только в Render → Environment.
> В `.gitignore` уже стоят `.env` и `*.json`.

### Шаг 3а. Деплой через Blueprint (проще — Render читает `render.yaml`)

1. Render → **New → Blueprint**
2. Подключи репозиторий `smmsemmers/pastila_bot`
3. Render увидит `render.yaml` и предложит создать сервис `pastila-task-bot` (тип **worker**,
   план **starter** — уже прописано в файле)
4. Заполни значения переменных с `sync: false` (`BOT_TOKEN`, `GROUP_CHAT_ID` и т.д.)
5. **Apply** → через минуту бот оживёт

### Шаг 3б. Деплой вручную (если не через Blueprint)

1. Render → **New → Background Worker** (НЕ Web Service — у бота нет порта)
2. Подключи репозиторий, ветка `main`
3. **Instance Type: Starter** (для 24/7)
4. Build Command: `pip install -r requirements.txt`
5. Start Command: `python bot.py`
6. **Environment** → добавь переменные из Шага 2
7. **Create Background Worker**

### Проверка

Напиши боту `/start` в личку или `/id` в группе. Логи: Render → `pastila-task-bot` → **Logs**.

## 💻 Бот 2: @pastila_code_remote_bot

Это **Claude Code Remote** — Telegram-мост к [Claude Code](https://claude.com/claude-code).
Важно понимать его природу, потому что деплой отличается от обычного бота:

### Вариант A. Это стандартный Claude Code Remote (плагин)

Тогда «бот» — это не отдельная программа, а сам Claude Code, запущенный командой:

```bash
cd ~/pastila_bot
claude --channels plugin:telegram@claude-plugins-official
```

Он живёт, **пока запущен `claude` на включённой машине** с твоим доступом к Anthropic.
Его нельзя «задеплоить воркером» в привычном смысле: ему нужны Claude Code CLI, авторизация,
рабочая копия репозитория и бюджет на токены. Держать его 24/7 на Render — это отдельная
нетривиальная задача (свой Docker-образ с Node + Claude Code + `ANTHROPIC_API_KEY`). Если
хочешь именно это — скажи, распишу отдельно. Обычно его держат на своём компьютере/сервере.

### Вариант B. У тебя в локальной папке — свой отдельный код второго бота

Если второй бот — это **твой собственный** скрипт (свой `bot.py`/`main.py`), а не плагин
Claude Code, то запушить его из **облачной сессии Claude нельзя** — она видит только этот
репозиторий, до папок на твоём компьютере не дотягивается. Пуши **из локального терминала**:

```bash
cd ~/путь-к-папке-второго-бота      # папка со вторым ботом
git init                             # если ещё не git-репозиторий
git add -A
git commit -m "Второй бот: первый коммит"

# создай пустой репозиторий на github.com (напр. pastila_code_remote_bot), затем:
git remote add origin https://github.com/smmsemmers/ИМЯ-РЕПО.git
git branch -M main
git push -u origin main
```

Дальше — как Бот 1: Render → **New → Background Worker** (Starter для 24/7) → подключи
этот новый репозиторий → задай Build/Start команды и переменные окружения второго бота.

> 💡 Альтернатива: если хочешь один Render-Blueprint на оба бота — можно положить код
> второго бота в подпапку этого репозитория и добавить в `render.yaml` второй сервис с
> `rootDir`. Скажи — сделаю, когда код второго бота будет доступен.

## 💸 Хочу бесплатно (временный вариант, без гарантии 24/7)

Бесплатного worker'а нет, поэтому бесплатно бот можно поднять только как **web-сервис** —
но для этого ему нужен открытый HTTP-порт (сейчас в `bot.py` его нет, нужна маленькая
доработка: поднять фоновый health-эндпоинт, отдающий `200 OK`). И даже так:

- сервис **засыпает** через 15 мин простоя → расписание (дайджест/алерты) будет пропускаться;
- два бота 24/7 не влезут в лимит 750 ч/мес.

Если готов мириться с этим ради экономии — скажи, добавлю health-порт в код и распишу
настройку free web-сервиса. Для стабильной работы всё же рекомендую Starter.

## 🧭 Частые проблемы

| Симптом | Причина / решение |
|---------|-------------------|
| В Render нельзя выбрать план Free для воркера | Так и есть — воркеры только платные. Бери Starter. |
| Бот засыпает, дайджест не приходит | Это free web-сервис. Нужен платный worker. |
| Все free-сервисы вдруг отключились | Превышен лимит 750 ч/мес. Держи не больше одного сервиса 24/7 на free. |
| Бот в логах крутит рестарт с `RuntimeError: no current event loop` | Python 3.14. Закреплён 3.12 через `.python-version` и `PYTHON_VERSION`. |
| Таск публикуется, но не пишется в таблицу | Сервисному email из `GOOGLE_CREDENTIALS` не выдан доступ к таблице (Редактор). |
